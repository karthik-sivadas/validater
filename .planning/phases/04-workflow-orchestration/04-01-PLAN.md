  <action>
Create `packages/db/src/schema/test-runs.ts` with three tables following the existing drizzle-orm/pg-core conventions in users.ts:

1. **testRunStatusEnum** - pgEnum('test_run_status') with values: 'pending', 'crawling', 'generating', 'validating', 'executing', 'persisting', 'complete', 'failed'

2. **testRuns** - pgTable('test_runs'):
   - id: text('id').primaryKey() -- nanoid, also used as Temporal workflowId
   - userId: text('user_id').notNull().references(() => users.id, { onDelete: 'cascade' })
   - url: text('url').notNull()
   - testDescription: text('test_description').notNull()
   - status: testRunStatusEnum('status').notNull().default('pending')
   - viewports: jsonb('viewports').notNull() -- array of viewport name strings
   - error: text('error') -- nullable, populated on failure
   - createdAt: timestamp('created_at').notNull().defaultNow()
   - updatedAt: timestamp('updated_at').notNull().defaultNow()
   - completedAt: timestamp('completed_at') -- nullable

3. **testRunResults** - pgTable('test_run_results'):
   - id: text('id').primaryKey()
   - testRunId: text('test_run_id').notNull().references(() => testRuns.id, { onDelete: 'cascade' })
   - viewport: text('viewport').notNull()
   - url: text('url').notNull()
   - totalDurationMs: integer('total_duration_ms').notNull()
   - startedAt: timestamp('started_at').notNull()
   - completedAt: timestamp('completed_at').notNull()
   - createdAt: timestamp('created_at').notNull().defaultNow()

4. **testRunSteps** - pgTable('test_run_steps'):
   - id: text('id').primaryKey()
   - resultId: text('result_id').notNull().references(() => testRunResults.id, { onDelete: 'cascade' })
   - stepId: text('step_id').notNull() -- references TestStep.id
   - stepOrder: integer('step_order').notNull()
   - status: text('status').notNull() -- 'pass' | 'fail'
   - errorMessage: text('error_message')
   - errorExpected: text('error_expected')
   - errorActual: text('error_actual')
   - screenshotBase64: text('screenshot_base64')
   - durationMs: integer('duration_ms').notNull()

Import { users } from './users.js' for the foreign key reference. Import { pgTable, text, timestamp, integer, jsonb, pgEnum } from 'drizzle-orm/pg-core'.

Update `packages/db/src/schema/index.ts` to add: `export * from "./test-runs.js";`

  <action>
Update both activities to use the browser pool singleton instead of launching standalone chromium instances. Follow the exact pattern from `execute-steps.activity.ts`:

**crawl-dom.activity.ts:**
- Remove `import { chromium } from 'playwright'`
- Add `import { getDefaultPool } from '../browser/pool.js'`
- Replace `chromium.launch()` with `const pool = getDefaultPool(); const pooled = await pool.acquire();`
- Use `pooled.browser.newContext()` then `context.newPage()` instead of `browser.newPage()` directly. Use a default viewport context (no specific viewport settings needed for crawling).
- Wrap in nested try/finally: outer for pool release (`pooled.pagesProcessed++; await pool.release(pooled)`), inner for context close (`await context.close()`)
- Remove `await browser.close()` -- pool manages browser lifecycle
- Keep the same function signature and return type

**validate-steps.activity.ts:**
- Same pattern: remove chromium import, add pool import
- Acquire from pool, create context, get page from context
- Use context.newPage() (default viewport is fine for validation)
- Navigate to URL on the context page
- Run validation and healing on the page
- Nested try/finally for context close and pool release
- Keep the same function signature and return type

Both activities should follow the established pattern: `pool.acquire() -> try { browser.newContext() -> try { context.newPage() -> work } finally { context.close() } } finally { pooled.pagesProcessed++; pool.release(pooled) }`
