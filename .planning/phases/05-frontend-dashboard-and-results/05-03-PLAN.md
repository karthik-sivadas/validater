  <action>
    Create directory: `packages/web/src/routes/_authed/runs/` (mkdir -p).

    Create `packages/web/src/routes/_authed/runs/$runId.tsx`:

    **Imports:**
    - `createFileRoute`, `Link` from `@tanstack/react-router`
    - `getTestRunDetail` from `@/server/test-runs`
    - shadcn: `Card`, `CardContent`, `CardHeader`, `CardTitle`, `Badge`, `Tabs`, `TabsContent`, `TabsList`, `TabsTrigger`, `ScrollArea`, `Dialog`, `DialogContent`, `DialogTrigger`
    - `useState` from `react`

    **Route definition:**
    ```typescript
    export const Route = createFileRoute("/_authed/runs/$runId")({
      loader: async ({ params }) => {
        const detail = await getTestRunDetail({ data: { testRunId: params.runId } });
        if (!detail) throw new Error("Test run not found");
        return detail;
      },
      component: TestRunDetailPage,
    });
    ```

    **TestRunDetailPage component:**

    Get data: `const { run, results } = Route.useLoaderData();`
    State: `const [selectedScreenshot, setSelectedScreenshot] = useState<string | null>(null);`

    **Layout structure (top to bottom):**

    1. **Header section:**
       - Back link: `<Link to="/runs" className="text-sm text-muted-foreground hover:text-foreground">` with "Back to History" text
       - Title: "Test Run" with run.id in muted small text
       - URL: show `run.url` as clickable link (target="_blank")
       - Description: show `run.testDescription` in a muted paragraph
       - Status Badge: map run.status to appropriate variant (same mapping as dashboard progress):
         - "complete" -> default (green-ish), "failed" -> destructive, others -> secondary
       - Timestamps: "Created {formatted date}" and if completedAt, "Completed {formatted date}". Use `new Intl.DateTimeFormat('en-US', { dateStyle: 'medium', timeStyle: 'short' }).format(new Date(timestamp))`.
       - If run.error, show error text in destructive color

    2. **Inline report summary (before viewport tabs):**
       - A Card with title "Summary"
       - Show total viewports tested: `{results.length} viewport(s)`
       - For each result: a row showing viewport name, total step count, pass count, fail count, duration in seconds (totalDurationMs / 1000 formatted to 1 decimal)
       - Calculate overall: total steps across all viewports, total passed, total failed
       - Show overall pass rate as percentage: `{Math.round((totalPassed / totalSteps) * 100)}% passed`
       - Use green text for 100% pass, yellow for >50%, red for <=50%

    3. **Viewport Tabs:**
       - `<Tabs defaultValue={results[0]?.viewport || "desktop"}>`
       - `<TabsList>` with a `<TabsTrigger>` for each result's viewport name
       - Each TabsTrigger shows: viewport name + a small pass/fail indicator (e.g., green dot for all pass, red dot if any fail)

    4. **TabsContent for each viewport:**
       - Show viewport metadata: viewport name, totalDurationMs as "X.Xs", started/completed timestamps
       - **Step list:** Use a `<ScrollArea className="h-[600px]">` if more than 5 steps, otherwise just render directly
       - For each step in `result.steps`, ordered by stepOrder:
         - A Card (or div with border) containing:
           - **Step header:** "Step {stepOrder}" with status Badge ("pass" -> default green, "fail" -> destructive)
           - **Step ID** in muted small text
           - **Duration:** "{durationMs}ms"
           - **Error details (if step.status === "fail"):**
             - errorMessage in destructive text
             - If errorExpected and errorActual: show "Expected: {errorExpected}" and "Actual: {errorActual}" in a code-like block (bg-muted, rounded, p-2, font-mono, text-sm)
           - **Screenshot:** If step.screenshotBase64 exists and is non-empty:
             - Render as `<img src={\`data:image/png;base64,${step.screenshotBase64}\`} alt={\`Step ${step.stepOrder} screenshot\`} className="w-full max-w-2xl rounded-md border border-border cursor-pointer" loading="lazy" />`
             - Wrap in a `<Dialog>` for zoom: clicking the thumbnail opens a Dialog with the full-size screenshot
             - DialogContent should have `className="max-w-4xl"` and the full-size img inside
           - If screenshotBase64 is empty/null, show a muted "No screenshot available" text

    **Screenshot zoom Dialog:**
    Use `selectedScreenshot` state. When user clicks a step screenshot, set it. Dialog open state is `!!selectedScreenshot`, onOpenChange sets to null.

    **IMPORTANT anti-patterns to avoid:**
    - Do NOT load all screenshots at once in the DOM. Use `loading="lazy"` on all img tags.
    - Show one viewport at a time via Tabs (not all viewports simultaneously).
    - Use `Intl.DateTimeFormat` for dates, NOT manual string manipulation.
