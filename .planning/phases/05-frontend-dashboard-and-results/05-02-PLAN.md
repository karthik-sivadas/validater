  <name>Task 1: Create useTestRunPolling hook</name>
  <files>packages/web/src/hooks/use-test-run-polling.ts</files>
  <action>
    Create `packages/web/src/hooks/use-test-run-polling.ts`:

    This hook polls `getTestRunStatusFn` at a configurable interval, stopping when the run reaches a terminal state.

    ```typescript
    import { useState, useEffect, useRef, useCallback } from "react";
    import { getTestRunStatusFn } from "@/server/run-test";
    ```

    **Interface:**
    ```typescript
    export function useTestRunPolling(testRunId: string | null, intervalMs = 2000)
    ```

    **State:**
    - `status` state: holds the latest status object `{ phase, viewportsComplete, viewportsTotal, error? }` or null
    - `isPolling` state: boolean indicating if polling is active
    - `intervalRef` as useRef for the setInterval handle

    **Behavior:**
    - If `testRunId` is null, do nothing (return null status)
    - On mount / testRunId change:
      1. Immediately call `getTestRunStatusFn({ data: { testRunId } })`
      2. Set up `setInterval` at `intervalMs` to poll
      3. On each poll response: update status. If `result.found && result.status`, set status. If `status.phase === "complete" || status.phase === "failed"`, clear interval and set isPolling=false.
    - Return `{ status, isPolling, stopPolling }` where stopPolling is a stable callback via useCallback
    - Cleanup: return a function from useEffect that clears the interval

    **IMPORTANT:** The `stopPolling` callback must be defined before the useEffect that uses it. Use `useCallback` wrapping `clearInterval(intervalRef.current)`.

    Terminal phases: `"complete"`, `"failed"`.
  </action>

  </verify>

    Completely rewrite `packages/web/src/routes/_authed/dashboard.tsx`. Remove the current placeholder card. The page has two states: (1) form state for creating a test, (2) progress state showing execution status.

    **Imports needed:**
    - `createFileRoute`, `useNavigate` from `@tanstack/react-router`
    - `useState` from `react`
    - `runTest` from `@/server/run-test`
    - `useTestRunPolling` from `@/hooks/use-test-run-polling`
    - shadcn components: `Button`, `Card`/`CardContent`/`CardHeader`/`CardTitle`, `Input`, `Textarea`, `Label`, `Progress`, `Badge`

    **Route definition:**
    ```typescript
    export const Route = createFileRoute("/_authed/dashboard")({
      component: DashboardPage,
    });
    ```

    **DashboardPage component:**

    State:
    - `url` (string, default "")
    - `testDescription` (string, default "")
    - `isSubmitting` (boolean, default false)
    - `testRunId` (string | null, default null)
    - `error` (string | null, default null)

    Hook: `const { status, isPolling } = useTestRunPolling(testRunId);`
    Also: `const navigate = useNavigate();`

    **Form state (when testRunId is null):**
    - A Card with title "Create New Test"
    - URL input: `<Input type="url" placeholder="https://example.com" value={url} onChange={...} />` with Label "Target URL"
    - Test description: `<Textarea placeholder="Describe what to test..." value={testDescription} onChange={...} rows={4} />` with Label "Test Description"
    - A brief helper text under the textarea: "Describe the user journey to test in plain English"
    - Submit button: `<Button onClick={handleSubmit} disabled={isSubmitting || !url || !testDescription}>` text "Run Test" (or "Starting..." when isSubmitting)
    - If error state is set, show a red text message: `<p className="text-sm text-destructive">{error}</p>`

    **handleSubmit function:**
    ```
    setIsSubmitting(true); setError(null);
    try {
      const result = await runTest({ data: { url, testDescription } });
      setTestRunId(result.testRunId);
    } catch (e) {
      setError(e instanceof Error ? e.message : "Failed to start test");
    } finally {
      setIsSubmitting(false);
    }
    ```

    **Progress state (when testRunId is not null):**
    - A Card with title "Test Run in Progress"
    - Show test run ID in muted text (truncated or small font)
    - Display current phase as a Badge: map phase to human-readable label and color variant:
      - pending -> "Pending" (outline)
      - crawling -> "Crawling Page" (secondary)
      - generating -> "Generating Steps" (secondary)
      - validating -> "Validating Locators" (secondary)
      - executing -> "Executing Tests" (secondary)
      - persisting -> "Saving Results" (secondary)
      - complete -> "Complete" (default/primary)
      - failed -> "Failed" (destructive)
    - Progress bar: `<Progress value={progressPercent} />` where progressPercent is calculated:
      - If status is null or pending: 0
      - Map phases to approximate progress: crawling=15, generating=30, validating=50, executing=70, persisting=85, complete=100, failed=100 (or use viewportsComplete/viewportsTotal for executing phase)
    - Viewport progress: "{viewportsComplete} of {viewportsTotal} viewports complete" text
    - If status?.error, show error text in destructive color
    - When status?.phase === "complete": show a "View Results" Button that navigates to `/runs/${testRunId}` using `navigate({ to: "/runs/$runId", params: { runId: testRunId } })`
    - When complete or failed, show a "New Test" Button (variant="outline") that resets all state back to form (setTestRunId(null), setUrl(""), setTestDescription(""), setError(null))

    **Layout:** Center the card on the page using flex, max-w-xl, consistent with the existing dashboard style.
