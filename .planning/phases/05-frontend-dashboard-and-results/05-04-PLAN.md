    **Search schema:**
    ```typescript
    const historySearchSchema = z.object({
      page: fallback(z.number().int().positive(), 1),
      pageSize: fallback(z.number().int().positive(), 20),
      status: fallback(z.enum(["pending", "complete", "failed", "all"]), "all"),
      search: z.string().optional(),
    });
    ```

    **Route definition:**
    ```typescript
    export const Route = createFileRoute("/_authed/runs/")({
      validateSearch: zodValidator(historySearchSchema),
      loaderDeps: ({ search }) => search,
      loader: async ({ deps }) => {
        return getTestRunList({ data: deps });
      },
      component: TestHistoryPage,
    });
    ```

    **TestHistoryPage component:**

    Get data: `const data = Route.useLoaderData();`
    Get search: `const search = Route.useSearch();`
    Get navigate: `const navigate = useNavigate();`
    Local state: `const [searchInput, setSearchInput] = useState(search.search || "");`

    **Helper: updateSearch function** that calls `navigate({ search: (prev) => ({ ...prev, ...updates }) })` -- this merges new search params into existing ones.

    **Layout (top to bottom):**

    1. **Page header:**
       - Title: "Test History"
       - Subtitle: "{data.total} test run(s)" in muted text

    2. **Filter bar (flex row, gap-3, items-center):**
       - **Search input:** `<Input placeholder="Search by URL..." value={searchInput} onChange={(e) => setSearchInput(e.target.value)} onKeyDown={(e) => { if (e.key === "Enter") updateSearch({ search: searchInput || undefined, page: 1 }); }} className="max-w-sm" />`
       - A search Button (variant="outline", size="sm") that triggers the search (same as Enter key)
       - **Status filter:** Use shadcn `<Select value={search.status || "all"} onValueChange={(val) => updateSearch({ status: val as any, page: 1 })}>` with options: "All Statuses", "Pending", "Complete", "Failed"
       - If any filter is active (search or status !== "all"), show a "Clear Filters" Button (variant="ghost", size="sm") that resets search and status

    3. **Table:**
       - Use shadcn Table components (Table, TableHeader, TableRow, TableHead, TableBody, TableCell)
       - Columns: URL, Description, Status, Viewports, Created, (Action)
       - **URL column:** Truncate to ~50 chars with ellipsis. Wrap in a Tooltip showing full URL.
       - **Description column:** Truncate to ~60 chars with ellipsis
       - **Status column:** Badge with same variant mapping as other pages:
         - "complete" -> default
         - "failed" -> destructive
         - "pending"/"crawling"/"generating"/"validating"/"executing"/"persisting" -> secondary
       - **Viewports column:** show as comma-joined list from the viewports jsonb (cast as string[])
       - **Created column:** formatted date using `new Intl.DateTimeFormat('en-US', { dateStyle: 'medium', timeStyle: 'short' }).format(new Date(run.createdAt))`
       - **Action column:** `<Link to="/runs/$runId" params={{ runId: run.id }}>` styled as a Button (variant="ghost", size="sm") with text "View"
       - Each TableRow should also be clickable: `onClick={() => navigate({ to: "/runs/$runId", params: { runId: run.id } })}` with `className="cursor-pointer hover:bg-muted/50"`

    4. **Empty state:**
       - If `data.runs.length === 0` and no filters active: show a centered message "No test runs yet" with a link "Create your first test" pointing to /dashboard
       - If `data.runs.length === 0` and filters active: show "No test runs match your filters" with a "Clear Filters" button

    5. **Pagination controls (flex row, justify-between, items-center, mt-4):**
       - Left side: "Page {data.page} of {data.totalPages}" text
       - Right side: "Previous" and "Next" Buttons
       - Previous: disabled when page === 1, onClick: `updateSearch({ page: search.page - 1 })`
       - Next: disabled when page >= totalPages, onClick: `updateSearch({ page: search.page + 1 })`

    **IMPORTANT patterns:**
    - All filter/pagination state lives in URL search params via validateSearch -- NO local state for filters except the search input (which syncs to URL on Enter/click)
    - Use `fallback()` from @tanstack/zod-adapter for all params with defaults
    - The Select component from shadcn uses a specific API: `<Select value={...} onValueChange={...}>`. Check the actual installed select.tsx for the exact component names and props.
