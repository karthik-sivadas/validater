  <action>
    1. Create `packages/web/src/server/api-keys.ts` with three server functions:
       - `createApiKeyFn`: createServerFn that calls the Better Auth API key creation endpoint.
         Use `authClient.apiKey.create({ name })` (from the apiKeyClient plugin) OR use the server-side
         `auth.api.createApiKey({ body: { name }, headers: request.headers })` to create a key for
         the current session user. Accept `{ name: string }` input validated with Zod (name: min 1, max 100).
         Return the full API key value (only shown once at creation time).
       - `listApiKeysFn`: createServerFn that lists all API keys for the current user.
         Use `auth.api.listApiKeys({ headers: request.headers })` to get keys.
         Return array of `{ id, name, createdAt, expiresAt, lastUsedAt }` (never return the key value).
       - `revokeApiKeyFn`: createServerFn that revokes/deletes an API key by ID.
         Accept `{ keyId: string }` input. Use `auth.api.deleteApiKey({ body: { keyId }, headers: request.headers })`.
         Verify the key belongs to the current session user before deletion.

       IMPORTANT: These server functions use session-based auth (same as runTest, getTestRunStatusFn).
       They are for the web UI, NOT for API consumers. Use `getWebRequest()` from vinxi/http or
       the request from server function context to forward session cookies to Better Auth.

       GOTCHA: Better Auth API key plugin methods may use different names than shown above. Check the
       actual TypeScript types on `auth.api` after the plugin is configured (Task 1). The plugin
       typically exposes create, list, delete, and verify methods. Use whatever the actual API is.

    2. Create `packages/web/src/routes/_authed/settings.tsx`:
       - Use `createFileRoute('/_authed/settings')` with a component and loader
       - Loader: Call `listApiKeysFn()` to load existing keys
       - Component structure:
         a. Page title "Settings" with "API Keys" section
         b. "Create API Key" button that opens a dialog/form with a name input
         c. On creation, show the key value ONCE in a copy-to-clipboard display with a warning
            ("This key will only be shown once. Copy it now.")
         d. Table/list of existing keys showing: name, created date, last used date, and a
            "Revoke" button per key
         e. Revoke button triggers `revokeApiKeyFn({ keyId })` and refreshes the list
       - Use existing shadcn components: Button, Input, Dialog (install Dialog if not present:
         `npx shadcn@latest add dialog`), Table (install if not present: `npx shadcn@latest add table`)
       - Copy-to-clipboard: Use `navigator.clipboard.writeText()` with a toast notification (sonner)
       - After creation, invalidate/refetch the key list by calling `router.invalidate()` or
         re-calling the loader

    3. Update `packages/web/src/routes/_authed.tsx`:
