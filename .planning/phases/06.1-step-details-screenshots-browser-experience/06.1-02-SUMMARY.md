**Canvas-based live viewer with color-coded action badges, human-readable descriptions, and compact screenshot thumbnails across live and results views**

## Accomplishments
- Live viewer frame display replaced with canvas double-buffered rendering (BrowserCanvas component) -- eliminates flicker on frame updates
- StepEvent enriched with action and description fields for live streaming display
- Step rows in live viewer now show color-coded action badges (click=blue, fill=purple, assert=amber, navigate=green, etc.) and truncated description text
- Results page StepCard shows action badge between step number and status, with description fallback to stepId for old runs
- Screenshot thumbnails compact to max-h-48 with object-cover, preserving click-to-expand full-size Dialog zoom

## Files Created/Modified

- `packages/web/src/components/live-viewer.tsx` - Canvas-based BrowserCanvas component, ACTION_COLORS map, enriched StepRow with action badge and description
- `packages/web/src/routes/_authed/runs/$runId.tsx` - Nullable action/description in all step type interfaces, ACTION_COLORS map, action badge in StepCard, description display, compact screenshot thumbnails

## Decisions Made
- Used canvas with offscreen Image.onload pattern for flicker-free frame rendering -- only resizes canvas when image dimensions change
- Duplicated ACTION_COLORS in both files rather than extracting to shared module -- the map is small (8 entries) and both files already import from different packages
- Made action/description nullable (string | null) in results page types for backward compatibility with existing test runs that lack this data
- Set screenshot thumbnail to max-h-48 with object-cover for compact display; existing Dialog zoom still shows full-size on click
