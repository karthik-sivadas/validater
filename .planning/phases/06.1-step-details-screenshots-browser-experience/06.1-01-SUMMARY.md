## Decisions Made
- Used `onConflictDoNothing()` instead of `onConflictDoUpdate()` for screenshot staging -- simpler and functionally equivalent since the first write wins (retry scenarios produce same screenshot data)
- Set `screenshotBase64: ''` instead of stripping the field from lightResults to maintain StepResult type compatibility without needing type casts
- Updated the dead `execute-viewports.activity.ts` file (superseded by child workflow pattern) to use factory pattern so it compiles
