    Create the core execution logic -- the 3 remaining files in `packages/core/src/execution/` plus barrel exports:

    **1. `execution/assertions.ts`** -- Assertion checker:
    - Create `AssertionError` class extending `Error` with `assertionType: string`, `expected: string`, `actual: string` properties. Constructor: `super(\`Assertion failed [\${assertionType}]: expected "\${expected}", got "\${actual}"\`)`. Set `this.name = 'AssertionError'`.
    - Create `async function checkAssertion(page: Page, step: TestStep, locator?: Locator): Promise<void>` that:
      - Returns immediately if `step.assertion` is undefined.
      - Handles page-level assertions (type `url`): compares `page.url()` with `assertion.expected` using `.includes()`.
      - Handles locator-level assertions (all others): requires locator parameter, throws if missing.
        - `visible`: `await locator.isVisible()` -- throws if false
        - `hidden`: `await locator.isHidden()` -- throws if false
        - `text`: `await locator.textContent()` -- throws `AssertionError` if text doesn't include expected
        - `value`: `await locator.inputValue()` -- throws `AssertionError` if value !== expected
        - `count`: `await locator.count()` -- throws `AssertionError` if count !== parseInt(expected)
        - `attribute`: parse expected as `"attrName=expectedValue"`, use `locator.getAttribute(attrName)`, throw `AssertionError` if mismatch
    - Import types from playwright (type-only) and from `../types/index.js`.

    **2. `execution/step-runner.ts`** -- Low-level step execution:
    - Create `async function resolveLocator(page: Page, step: TestStep): Promise<Locator>`:
      - Try primary locator first via `mapLocatorToPlaywright(page, locators[primaryLocatorIndex])`.
      - Check `await pwLocator.count()` -- if > 0, return it.
      - On failure or count === 0, try alternatives sorted by confidence descending (filter out primaryLocatorIndex first).
      - Each alternative: try/catch wrapping `mapLocatorToPlaywright` + `.count()`. If count > 0, return.
      - If ALL fail, throw Error with descriptive message listing all tried locators.
      - Import `mapLocatorToPlaywright` from `../locators/mapper.js`.

    - Create `async function executeAction(locator: Locator, step: TestStep, timeout: number): Promise<void>`:
      - Switch on `step.action`:
        - `click`: `locator.click({ timeout })`
        - `fill`: `locator.fill(step.value ?? '', { timeout })`
        - `select`: `locator.selectOption(step.value ?? '', { timeout })`
        - `check`: `locator.check({ timeout })`
        - `hover`: `locator.hover({ timeout })`
        - `wait`: `locator.waitFor({ state: 'visible', timeout })`
        - `navigate`, `assert`: no-op (handled by caller)

    - Create `async function executeStep(page: Page, step: TestStep, config: ExecutionConfig): Promise<StepResult>`:
      - Record `startTime = Date.now()`
      - Initialize `status: 'pass'`, `error: undefined`, `locator: undefined`
      - Try block:
        - If `step.action === 'navigate'`: `page.goto(step.value ?? step.target.elementDescription, { waitUntil: 'networkidle', timeout: config.navigationTimeoutMs ?? 30_000 })`
        - Else if `step.action !== 'assert'` (action steps): resolve locator via `resolveLocator(page, step)`, then `executeAction(locator, step, config.stepTimeoutMs ?? 10_000)`
        - Else (pure assert): resolve locator via `resolveLocator(page, step)` (needed for locator-level assertions)
        - If `step.assertion` exists: `checkAssertion(page, step, locator)`
      - Catch block: set `status = 'fail'`, build `StepError` with message from caught error. If error is `AssertionError`, populate `expected` and `actual` from it.
      - Finally (ALWAYS): capture screenshot via `page.screenshot({ type: 'png', fullPage: config.screenshotFullPage ?? false })`, convert to base64 via `screenshot.toString('base64')`. Wrap screenshot in try/catch -- if screenshot fails (page crashed), use empty string.
      - Return `StepResult` with all fields.

    **3. `execution/step-executor.ts`** -- Orchestration loop:
    - Create `async function executeSteps(page: Page, steps: TestStep[], config?: ExecutionConfig): Promise<StepResult[]>`:
      - Default config: `{ stepTimeoutMs: 10_000, navigationTimeoutMs: 30_000, screenshotFullPage: false }`
      - Iterate over steps sequentially (NOT parallel -- browser has one page state).
      - For each step, call `executeStep(page, step, mergedConfig)`.
      - Push result to results array.
      - IMPORTANT: Continue executing ALL remaining steps even after a failure. Do NOT break or return early. Users need to see which steps are independently valid vs cascade failures.
      - Return the full `StepResult[]`.
