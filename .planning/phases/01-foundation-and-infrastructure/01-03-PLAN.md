<task type="auto">
  <name>Task 2: Drizzle schema, client, and migration</name>
  <files>
    packages/db/src/client.ts

    packages/db/src/schema/index.ts
    packages/db/drizzle.config.ts

  <action>
1. Create `packages/db/drizzle.config.ts`:
   ```typescript
   import "dotenv/config";
   import { defineConfig } from "drizzle-kit";

   export default defineConfig({
     out: "./src/migrations",
     schema: "./src/schema/index.ts",
     dialect: "postgresql",
     dbCredentials: {
       url: process.env.DATABASE_URL!,
     },
   });
   ```

2. Create `packages/db/src/schema/users.ts` with Better Auth tables. Use the exact schema from the research document: users, sessions, accounts, verifications tables. All use `pgTable` from `drizzle-orm/pg-core`. Use `text` for IDs (Better Auth generates string IDs). Include all columns: id, name, email, emailVerified, image, createdAt, updatedAt for users; and corresponding columns for sessions, accounts, verifications as shown in research.

3. Create `packages/db/src/schema/index.ts`:
   ```typescript
   export * from "./users";
   ```

4. Create `packages/db/src/client.ts`:
   ```typescript
   import "dotenv/config";
   import { drizzle } from "drizzle-orm/node-postgres";
   import { Pool } from "pg";
   import * as schema from "./schema";

   const pool = new Pool({
     connectionString: process.env.DATABASE_URL,
     max: 20,
   });

   export const db = drizzle(pool, { schema });
   export type Database = typeof db;
   ```

5. Replace `packages/db/src/index.ts` placeholder with:
   ```typescript
   export { db } from "./client";
   export type { Database } from "./client";
   export * from "./schema";
   ```

6. Install dependencies in packages/db if not already present: `pnpm --filter @validater/db install`

7. Push schema to database: `pnpm --filter @validater/db db:push`
   This creates all tables without generating migration files (faster for dev iteration).

8. Verify tables exist by running a quick query:
   ```bash
   docker compose -f docker/docker-compose.yml exec postgres psql -U validater -d validater -c "\dt"
   ```
   Should show users, sessions, accounts, verifications tables.
  </action>
  <verify>

  </verify>
