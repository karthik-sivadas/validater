  <action>
1. Install Better Auth in web package:
   ```bash
   pnpm --filter @validater/web add better-auth
   ```

2. Create `packages/web/src/lib/auth.ts` -- server-side Better Auth instance:
   ```typescript
   import { betterAuth } from "better-auth";
   import { drizzleAdapter } from "better-auth/adapters/drizzle";
   import { tanstackStartCookies } from "better-auth/tanstack-start";
   import { db } from "@validater/db";

   export const auth = betterAuth({
     database: drizzleAdapter(db, {
       provider: "pg",
     }),
     emailAndPassword: {
       enabled: true,
     },
     plugins: [tanstackStartCookies()], // MUST be last plugin
   });
   ```

3. Create `packages/web/src/lib/auth-client.ts` -- browser-side client:
   ```typescript
   import { createAuthClient } from "better-auth/react";

   export const authClient = createAuthClient({
     baseURL: import.meta.env.VITE_APP_URL || "http://localhost:3000",
   });
   ```

4. Create `packages/web/src/routes/api/auth/$.ts` -- catch-all API route handler:
   Follow the exact pattern from research. Creates a file route at `/api/auth/$` with GET and POST handlers that delegate to `auth.handler(request)`. This handles all Better Auth endpoints (sign-up, sign-in, sign-out, session).

5. Create `packages/web/src/server/auth.server.ts` -- session helper server functions:
   - `getSession`: createServerFn with method GET, uses `getRequestHeaders()` from `@tanstack/react-start/server`, calls `auth.api.getSession({ headers })`, returns session or null.
   - `ensureSession`: createServerFn with method GET, calls getSession internally, if no session throws redirect to `/login`.

6. Verify the auth module loads without errors: `pnpm --filter @validater/web exec -- node -e "console.log('auth module check')"` (basic sanity check that imports resolve).
  </action>

    Verify files exist: `ls packages/web/src/lib/auth.ts packages/web/src/lib/auth-client.ts packages/web/src/routes/api/auth/$.ts packages/web/src/server/auth.server.ts`
  </verify>
  <done>Better Auth server instance configured with Drizzle adapter and tanstackStartCookies plugin. API catch-all route handles all auth endpoints. Session helpers exported as server functions.</done>
