    The function should:
    1. Accept `page: Page` and `options: CrawlOptions` parameters

    Handle timeout errors gracefully -- wrap navigation in try/catch and throw a descriptive error: `new Error(\`Failed to crawl ${options.url}: ${err.message}\`)`.

  <action>
    **packages/core/src/dom/extractor.ts** -- Extract semantic and interactive elements from Cheerio-parsed HTML:

    `extractInteractiveElements($: cheerio.CheerioAPI): InteractiveElement[]`
    - Select all interactive elements: `a, button, input, select, textarea, [role="button"], [role="link"], [role="tab"], [role="menuitem"], [contenteditable="true"]`
    - For each element, extract:
      - `tag`, `type` (for inputs), `role` (aria role or implicit)
      - `name`: `aria-label` || `aria-labelledby` target text || visible text content (truncated to 100 chars)
      - `label`: associated `<label>` text (find by `for` attribute matching `id`, or parent label)
      - `placeholder`: placeholder attribute
      - `testId`: `data-testid` attribute
      - `text`: trimmed visible text content (truncated to 100 chars)
      - `value`: current value for inputs
      - `attributes`: all data-* and aria-* attributes
      - `xpath`: generate a simple XPath (e.g., `//button[contains(text(),"Submit")]` or `//input[@name="email"]`)
      - `cssSelector`: generate a unique CSS selector (prefer #id, then [data-testid], then tag.class:nth-of-type)
    - Return array sorted by DOM order

    `extractSemanticElements($: cheerio.CheerioAPI): SemanticElement[]`
    - Select landmark elements: `main, nav, header, footer, aside, section, article, form, h1-h6, [role="navigation"], [role="main"], [role="banner"]`
    - Extract tag, role, text (truncated), attributes
    - Build shallow tree structure (max 2 levels deep) to provide page layout context
    - Return array of semantic elements

    Helper: `isUtilityClass(className: string): boolean`
    - Return true for Tailwind-like patterns: classes starting with common prefixes (flex, grid, p-, m-, w-, h-, text-, bg-, border-, rounded-, shadow-, etc.) or containing colons (responsive modifiers like `md:`, `lg:`)
    - This strips noise from class attributes while keeping semantic class names

    **packages/core/src/dom/simplifier.ts** -- Orchestrate DOM simplification:

    `simplifyDom(html: string, options?: { maxTokenEstimate?: number, includeAriaSnapshot?: string }): SimplifiedDom`
    1. Load HTML with `cheerio.load(html)`
    2. Strip non-semantic elements: `script, style, noscript, svg, path, meta, link, head, iframe`
    3. Remove hidden elements: `[hidden], [aria-hidden="true"], [style*="display: none"], [style*="visibility: hidden"]`
    4. Strip all `style` attributes
    5. Clean class attributes: keep only non-utility classes (use isUtilityClass helper)
    6. Remove empty wrappers: `div:empty, span:empty` (iterate until stable -- removing empties can create new empties)
    7. Remove data attributes EXCEPT `data-testid`
    8. Call `extractInteractiveElements($)` and `extractSemanticElements($)`
    9. Estimate tokens: `Math.ceil($.html().length / 4)` (4 chars per token heuristic)
    10. If tokenEstimate > maxTokenEstimate (default 15000):
        - Progressive reduction: first remove non-interactive, non-landmark elements
        - Then truncate text nodes to 50 chars max
        - Then drop `attributes` from SemanticElements
        - Re-estimate tokens after each reduction step, stop when under budget
    11. Build and return `PageContext` from `title` tag text and landmark roles
    12. Return `SimplifiedDom` object

    **packages/core/src/dom/index.ts** -- Barrel export:


    **Important:** Use `.js` extensions on ALL relative imports. Import cheerio as `import * as cheerio from 'cheerio'`.
