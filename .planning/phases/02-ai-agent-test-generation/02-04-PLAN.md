    **packages/core/src/locators/validator.ts** -- Verify locators against a live Playwright Page:

    `verifyLocator(page: Page, locator: LocatorStrategy): Promise<LocatorVerification>`
    - Import Playwright `Page` type as type-only import
    - Map locator type to Playwright locator method:
      - `role`: Parse value as "roleName" or "roleName: accessible name" format. Use `page.getByRole(role, { name })` if name present, else `page.getByRole(role)`.
      - `text`: `page.getByText(value)`
      - `label`: `page.getByLabel(value)`
      - `placeholder`: `page.getByPlaceholder(value)`
      - `testId`: `page.getByTestId(value)`
      - `css`: `page.locator(value)`
      - `xpath`: `page.locator(\`xpath=${value}\`)`
    - Call `.count()` on the locator to check existence
    - Return `LocatorVerification` with found (count > 0), count, isUnique (count === 1)
    - Wrap in try/catch -- invalid selectors should return found: false, count: 0, not throw

    `verifyStepLocators(page: Page, step: TestStep): Promise<ValidationResult>`
    - Verify ALL locators in the step's target.locators array using `verifyLocator`
    - Determine if primary locator (at primaryLocatorIndex) is valid
    - Return ValidationResult with all locator results, primaryLocatorValid flag, and overall isValid (true if at least one locator found the element)

    **packages/core/src/locators/healer.ts** -- Self-healing when primary locator fails:

    `healLocator(page: Page, step: TestStep, validationResult: ValidationResult): Promise<LocatorStrategy | null>`
    - If primary locator failed but alternative locators succeeded:
      - Return the highest-confidence alternative locator that was verified successfully
      - No AI call needed -- just pick the best alternative
    - If ALL locators failed:
      - Use `buildValidationPrompt` from ai/prompts/templates.ts to construct a healing prompt
      - Call `generateObject` with a small schema (just a LocatorStrategy array) asking Claude to suggest new locators based on the step description and current page DOM
      - Verify the suggested locators against the live page
      - Return the first one that works, or null if none work
    - This is the fallback path -- most healing should happen via alternative locators (cheap), with AI healing as last resort (expensive)

    `healStepLocators(page: Page, steps: TestStep[], validationResults: ValidationResult[]): Promise<TestStep[]>`
    - For each step where primaryLocatorValid is false:
      - Call `healLocator` to find a working locator
      - If healed: update the step's primaryLocatorIndex to point to the working locator (or add the healed locator to the locators array)
      - If not healed: mark the step but don't remove it (downstream consumers decide what to do with unhealed steps)
    - Return the updated steps array

    **packages/core/src/generation/pipeline.ts** -- End-to-end orchestrator:

    `generateAndValidateTestSteps(params: { page: Page, request: GenerationRequest }): Promise<GenerationResult>`
    - This is the full pipeline in one function. Import Page as type-only.
    - Steps:
      1. **Crawl**: Call `crawlPage(page, { url: request.url, viewport: request.viewport })` to get raw HTML + aria snapshot
      2. **Simplify**: Call `simplifyDom(crawlResult.html, { maxTokenEstimate: request.maxTokenBudget || 15000, includeAriaSnapshot: crawlResult.ariaSnapshot })` to get token-efficient DOM
      3. **Generate**: Call `generateTestSteps({ simplifiedDomHtml: $.html(), ariaSnapshot: crawlResult.ariaSnapshot, testDescription: request.testDescription, model: request.model })` to get structured test steps
      4. **Assign IDs**: Use `nanoid()` to assign unique `id` to each step (the AI generates steps without IDs)
      5. **Validate**: Call `verifyStepLocators(page, step)` for each step to check locators against live page
      6. **Heal**: For any steps with failed primary locators, call `healStepLocators(page, steps, validationResults)` to attempt self-healing
      7. **Calculate cost**: Call `calculateCost(usage, model)` for cost tracking
      8. Return `GenerationResult` with steps, reasoning, usage, cost, model, and durationMs (track with `Date.now()` start/end)

    ```typescript
    import { chromium } from 'playwright';
    import { crawlPage, simplifyDom } from '@validater/core';
    import type { CrawlOptions, SimplifiedDom, CrawlResult } from '@validater/core';

    export async function crawlDom(options: CrawlOptions): Promise<{ crawlResult: CrawlResult; simplified: SimplifiedDom }> {
      const browser = await chromium.launch({ headless: true });
      try {
        const page = await browser.newPage();
        const crawlResult = await crawlPage(page, options);
        const simplified = simplifyDom(crawlResult.html, {
          maxTokenEstimate: 15000,
          includeAriaSnapshot: crawlResult.ariaSnapshot,
        });
        return { crawlResult, simplified };
      } finally {
        await browser.close();
      }
    }
    ```

    **packages/worker/src/activities/generate-steps.activity.ts** -- Temporal activity:
    ```typescript
    import { generateTestSteps, queuedRequest, defaultApiQueue, calculateCost } from '@validater/core';
    import type { GenerationResult, GenerationRequest } from '@validater/core';
    import { nanoid } from 'nanoid';

    export async function generateSteps(params: {
      simplifiedDomHtml: string;
      ariaSnapshot: string;
      testDescription: string;
      model?: string;
    }): Promise<{ steps: Array<{ id: string; order: number; action: string; description: string; target: any; value?: string; assertion?: any; reasoning: string }>; reasoning: string; usage: any; cost: any }> {
      const result = await queuedRequest(defaultApiQueue, () =>
        generateTestSteps(params)
      );
      // Assign unique IDs to each step
      const stepsWithIds = result.steps.map((step) => ({
        ...step,
        id: nanoid(),
      }));
      const cost = calculateCost(result.usage, params.model);
      return { steps: stepsWithIds, reasoning: result.reasoning, usage: result.usage, cost };
    }
    ```

    **packages/worker/src/activities/validate-steps.activity.ts** -- Temporal activity:
    ```typescript
    import { chromium } from 'playwright';
    import { verifyStepLocators, healStepLocators } from '@validater/core';
    import type { TestStep, ValidationResult } from '@validater/core';

    export async function validateSteps(params: {
      url: string;
      steps: TestStep[];
    }): Promise<{ validatedSteps: TestStep[]; validationResults: ValidationResult[] }> {
      const browser = await chromium.launch({ headless: true });
      try {
        const page = await browser.newPage();
        await page.goto(params.url, { waitUntil: 'networkidle', timeout: 30_000 });

        // Validate all steps
        const validationResults: ValidationResult[] = [];
        for (const step of params.steps) {
          const result = await verifyStepLocators(page, step);
          validationResults.push(result);
        }

        // Heal failed steps
        const healedSteps = await healStepLocators(page, params.steps, validationResults);

        return { validatedSteps: healedSteps, validationResults };
      } finally {
        await browser.close();
      }
    }
    ```
