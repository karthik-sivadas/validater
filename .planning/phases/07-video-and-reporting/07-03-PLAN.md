     ```tsx
     <Button
       variant="outline"
       size="sm"
       disabled={loadingVideo}
       onClick={async () => {
         setLoadingVideo(true);
         try {
           const result = await getVideoFile({ data: { testRunId: run.id, viewport: result.viewport } });
           if (result.found) {
             const binary = atob(result.videoBase64);
             const bytes = new Uint8Array(binary.length);
             for (let i = 0; i < binary.length; i++) bytes[i] = binary.charCodeAt(i);
             const blob = new Blob([bytes], { type: result.mimeType });
             setVideoUrl(URL.createObjectURL(blob));
           }
         } finally {
           setLoadingVideo(false);
         }
       }}
     >
       {loadingVideo ? 'Loading...' : 'Play Debug Video'}
     </Button>
     ```

         >
           <p className="text-sm text-muted-foreground p-4">

         </video>

3. **Create FFmpeg processor** at `packages/worker/src/video/processor.ts`:
   - Import `spawn` from `child_process`, `ffmpegPath` from `ffmpeg-static`.
   - Define helper `runFFmpeg(args: string[]): Promise<void>` that wraps spawn in a Promise (see research Pattern 3).
   - Export `processPolishedVideo(options: ProcessVideoOptions): Promise<void>`:
     ```typescript
     interface AnnotationOverlay {
       text: string;
       startTimeSec: number;
       endTimeSec: number;
     }
     interface ProcessVideoOptions {
       inputPath: string;
       outputPath: string;
       annotations: AnnotationOverlay[];
       resolution: { width: number; height: number };
       trimSegments?: Array<{ start: number; end: number }>;
     }
     ```
   - Build FFmpeg filter pipeline:
     - **Annotations**: Build `drawtext` filters for each annotation. Escape colons and single quotes in text. Use `enable='between(t,start,end)'` for timed display. Position at bottom of frame (`y=h-40`), white text on semi-transparent black box.
     - **Dead time trimming**: If `trimSegments` provided, use `trim`+`setpts`+`concat` filter chain. Each segment is extracted, PTS reset, then concatenated.
     - **Resolution**: Apply `scale={width}:{height}` filter.
     - If no trimming, use simple `-vf` with comma-separated filters.
     - If trimming, use `-filter_complex` with the full chain.
   - Output codec: `-c:v libx264 -preset fast -crf 23` (H.264 MP4, good balance of speed and quality).
   - Add `-y` flag to overwrite output.
   - Validate `ffmpegPath` is not null before spawning (throw descriptive error if ffmpeg-static didn't install correctly).

   - Export `processVideoActivity(params: ExportVideoParams): Promise<ExportVideoResult>`:
     ```typescript
     interface ExportVideoParams {
       testRunId: string;
       viewport: string;
       videoPath: string; // relative path from DB
       resolution: { width: number; height: number };
       includeAnnotations: boolean;
       trimDeadTime: boolean;

         durationMs: number;
       }>;
     }
     interface ExportVideoResult {
       outputPath: string; // relative path to processed MP4
     }
     ```
   - Import `getVideoPath` from `../video/storage.js` and `processPolishedVideo` from `../video/processor.js`.
   - Resolve the input video absolute path via `getVideoPath(params.videoPath)`.
   - Build output path: same directory as input but with `.mp4` extension and `-polished` suffix (e.g., `{testRunId}/{viewport}-polished.mp4`). Use `getVideoPath()` to resolve.
   - Build `annotations` array from `params.steps` if `includeAnnotations`:
     - Compute cumulative start time for each step based on step durations.
     - Each annotation: `text: "Step {stepOrder}: {action} - {description}"`, `startTimeSec` and `endTimeSec` from step timing.
   - Build `trimSegments` from step timing data if `trimDeadTime`:
     - Identify active segments (where steps are executing). Each step occupies `[cumStart, cumStart + durationMs/1000]`.
     - Add 0.5s padding before and after each step segment.
     - Merge overlapping/adjacent segments.
     - These become the `trimSegments` -- everything outside these segments is "dead time" that gets cut.
   - Call `processPolishedVideo({ inputPath, outputPath, annotations, resolution, trimSegments })`.
   - Return `{ outputPath: relativeOutputPath }`.
