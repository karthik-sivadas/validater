  <action>
  1. **Create `packages/web/src/hooks/use-live-stream.ts`:**
     Export `useLiveStream(testRunId: string | null)` hook returning `{ frame: string | null; steps: StepEvent[]; connected: boolean; ended: boolean }`

     Define local types (do NOT import from worker package -- these are client-side duplicates):
     - `StepEvent`: `{ stepId: string; stepOrder: number; status: 'pass' | 'fail'; durationMs: number; error?: string }`
     - `StreamMessage`: `{ type: 'frame' | 'step-start' | 'step-complete' | 'stream-end'; testRunId: string; timestamp: number; payload: unknown }`

     Implementation:
     - Use `useState` for frame (string | null), steps (StepEvent[]), connected (boolean), ended (boolean)
     - Use `useRef` for WebSocket instance and retry count
     - Use `useEffect` keyed on testRunId:
       - If testRunId is null, return early (no connection)
       - Define `connect()` function:
         - Create `new WebSocket(wsUrl)` where `wsUrl` is `${import.meta.env.VITE_WS_URL ?? 'ws://localhost:3001'}/stream/${testRunId}`
         - Use `import.meta.env.VITE_WS_URL` so the WS URL is configurable via env var
         - `onopen`: set connected=true, reset retry count to 0
         - `onmessage`: parse JSON as StreamMessage, handle by type:
           - `'frame'`: set frame to `(msg.payload as { data: string }).data`
           - `'step-complete'`: append payload (cast to StepEvent) to steps array
           - `'stream-end'`: set ended=true, close the WebSocket (do NOT reconnect on stream-end)
         - `onclose`: set connected=false. If NOT ended and retries < 10, reconnect with exponential backoff: delay = min(500 * 2^retries, 30000) + random(0, 500) jitter. Increment retry count. Use setTimeout to call connect().
         - `onerror`: just log to console.warn (onclose will handle reconnect)
       - Call `connect()`
       - Cleanup: close WebSocket on unmount
     - Reset all state (frame, steps, connected, ended) when testRunId changes

  2. **Create `packages/web/src/components/live-viewer.tsx`:**
     Export `LiveViewer` component accepting `{ testRunId: string }` props.

     Implementation:
     - Call `useLiveStream(testRunId)` to get frame, steps, connected, ended
     - Layout: `grid grid-cols-3 gap-4` (browser feed 2/3, step log 1/3)
     - Left panel (col-span-2):
       - Connection indicator: absolute positioned top-right, show a pulsing red dot with "LIVE" text when connected, show "Reconnecting..." when not connected and not ended, show "Stream ended" when ended
       - Frame display: If frame exists, render `<img src={`data:image/jpeg;base64,${frame}`} alt="Live browser view" className="w-full rounded-lg border" />` with `aspect-video` container
       - Placeholder: If no frame yet, show a muted `aspect-video` div with "Waiting for stream..." text centered
     - Right panel (col-span-1):
       - Header: "Step Log" with step count badge
       - ScrollArea (from shadcn) with max height ~500px
       - Render each step event as a row with:
         - Badge: green "pass" or red "fail" variant
         - "Step {stepOrder}" label
         - Duration in ms (text-xs text-muted-foreground)
         - If error, show error text below in red
       - Auto-scroll to bottom when new steps arrive (use useRef on scroll container, scrollIntoView on last element)
     - Import Badge from `@/components/ui/badge`, ScrollArea from `@/components/ui/scroll-area`
     - Import useLiveStream from `@/hooks/use-live-stream`
