  1. **Add Redis container to docker-compose.yml:**
     Add a `redis` service using `redis:7-alpine` image, mapping port 6379:6379, with a `redis_data` volume for persistence. Add `redis_data` to the volumes section.

  2. **Install streaming dependencies in packages/worker:**
     Run `cd packages/worker && pnpm add ioredis hono @hono/node-server @hono/node-ws`

  3. **Create `packages/worker/src/streaming/types.ts`:**
     Define shared types for the streaming pipeline:
     - `ScreencastFrame`: `{ data: string; sessionId: number; metadata: { offsetTop: number; pageScaleFactor: number; deviceWidth: number; deviceHeight: number; scrollOffsetX: number; scrollOffsetY: number; timestamp: number } }`
     - `StepEvent`: `{ stepId: string; stepOrder: number; status: 'pass' | 'fail'; durationMs: number; error?: string }`
     - `StreamMessage`: `{ type: 'frame' | 'step-start' | 'step-complete' | 'stream-end'; testRunId: string; timestamp: number; payload: unknown }`
     - `StreamingConfig`: `{ testRunId: string; enabled: boolean }`
     - Helper function `channelName(testRunId: string): string` returning `stream:${testRunId}`

  4. **Create `packages/worker/src/streaming/screencast.ts`:**
     Export `startScreencast(page: Page, onFrame: (frame: ScreencastFrame) => void): Promise<() => Promise<void>>`
     - Get CDPSession via `await page.context().newCDPSession(page)`
     - Listen to `Page.screencastFrame` event
     - CRITICAL: Acknowledge the frame IMMEDIATELY with `cdp.send('Page.screencastFrameAck', { sessionId: frame.sessionId })` BEFORE calling onFrame. This prevents backpressure stalling.
     - Call `onFrame(frame)` after acknowledgment
     - Start screencast with: `cdp.send('Page.startScreencast', { format: 'jpeg', quality: 60, maxWidth: 1280, maxHeight: 720, everyNthFrame: 2 })`
     - Return a cleanup function that calls `Page.stopScreencast` and `cdp.detach()`, wrapped in try/catch (page may already be closed)
     - Import Page type from `playwright` (type-only import)

  5. **Create `packages/worker/src/streaming/redis-publisher.ts`:**
     - Singleton pattern: `let _publisher: Redis | null = null`
     - `getRedisPublisher(): Redis` -- lazily creates ioredis connection to `process.env.REDIS_URL ?? 'redis://localhost:6379'`, attaches error handler that logs to console.error
     - `publishFrame(testRunId: string, frame: ScreencastFrame): Promise<void>` -- creates StreamMessage with type 'frame', payload is `{ data: frame.data }` (only base64 string, not metadata), publishes JSON to channelName(testRunId)
     - `publishStepEvent(testRunId: string, event: StepEvent): Promise<void>` -- creates StreamMessage with type 'step-complete', publishes to same channel
     - `publishStreamEnd(testRunId: string): Promise<void>` -- creates StreamMessage with type 'stream-end', publishes to same channel
     - `shutdownRedis(): Promise<void>` -- disconnects publisher, sets to null
     - Import Redis from `ioredis`, import types from `./types.js`
  </action>

  1. **Create `packages/worker/src/streaming/ws-sidecar.ts`:**
     Export `startWsSidecar(port?: number): void`
     - Create Hono app instance
     - Use `createNodeWebSocket({ app })` from `@hono/node-ws` to get `injectWebSocket` and `upgradeWebSocket`
     - Add CORS middleware: set `Access-Control-Allow-Origin` to `process.env.WS_CORS_ORIGIN ?? 'http://localhost:3000'` on all responses
     - Add health check route: `GET /health` returns `{ status: 'ok' }`
     - Add WebSocket route: `GET /stream/:testRunId` using `upgradeWebSocket((c) => { ... })`
       - `onOpen`: Create a NEW Redis subscriber connection (ioredis requires separate connections for pub/sub). Subscribe to `stream:${testRunId}`. On 'message' event, relay the message string directly to `ws.send(message)`.
       - `onClose`: Unsubscribe, disconnect the Redis subscriber, set to null.
       - `onError`: Same cleanup as onClose.
     - Call `serve({ fetch: app.fetch, port: port ?? 3001 })` from `@hono/node-server`, then `injectWebSocket(server)`
     - Log: `console.log('WebSocket sidecar started on port ${port ?? 3001}')`
     - IMPORTANT: Each WebSocket connection gets its OWN Redis subscriber instance. Do NOT share a single subscriber across connections.

  2. **Modify `packages/worker/src/activities/execute-steps.activity.ts`:**
     - Import `StreamingConfig` from `../streaming/types.js`
     - Import `startScreencast` from `../streaming/screencast.js`
     - Import `publishFrame`, `publishStepEvent`, `publishStreamEnd` from `../streaming/redis-publisher.js`
     - Add optional `streamingConfig?: StreamingConfig` field to `ExecuteStepsParams`
     - After `page.goto()` and before `executeSteps()`:
       - If `params.streamingConfig?.enabled`, call `startScreencast(page, (frame) => { publishFrame(params.streamingConfig!.testRunId, frame).catch(() => {}); })` and store the returned stop function
     - The core `executeSteps` function from `@validater/core` does NOT support callbacks. Instead, wrap the existing call: after `executeSteps` returns the stepResults array, iterate over stepResults and publish each as a step-complete event via `publishStepEvent`. This keeps the core package unchanged and avoids modifying the step executor.
     - After publishing step events, stop the screencast (call stored stop function)
     - After stopping screencast, call `publishStreamEnd(testRunId)` to signal the stream is done
     - Wrap all streaming operations in try/catch so streaming failures never break test execution. Streaming is best-effort.

  3. **Modify `packages/worker/src/workflows/viewport-execution.workflow.ts`:**

     - No other changes needed -- the workflow just proxies to the activity

  4. **Modify `packages/worker/src/workflows/test-run.workflow.ts`:**

     - Only enable streaming for the FIRST viewport to avoid multiple simultaneous streams overwhelming the client. Pass `enabled: true` only for index 0 in the `params.viewports.map()`, `enabled: false` for the rest.

  5. **Modify `packages/worker/src/worker.ts`:**
